## 1. User Intent Discovery

### 1-1. 핵심 문제
영업자들이 거래처별 단가표를 자기만 쥐고 있어, 경영진이 "우리 회사가 어디에 얼마에 팔고 있는지" 전체 그림을 모름. ERP에 견적 데이터(63,523건)가 있으니, 이걸로 거래처별 실제 적용 단가를 역산해보려는 것.

### 1-2. 대상 사용자
- **1차**: Jack (분석 수행자) + 경영진 (결과 소비자)
- **2차**: 추후 산출기 단가 엔진에 시드 데이터로 활용

### 1-3. 성공 기준
- 거래처 419곳 × 주요 항목(CTP, 인쇄 등)에 대해 실제 적용 단가 매트릭스가 추출됨
- 갑우 표준단가 대비 차이(%)가 표시됨
- HTML 대시보드에서 거래처별 필터링/정렬하며 탐색 가능

---

## 2. Alternatives Explored

### Approach A: 거래처×항목 2차원 매트릭스 — 선택됨
- **장점**: 단순하고 직관적. 419 거래처의 전체 그림이 한눈에 보임
- **단점**: 대수(R)에 따른 단가 변동을 반영하지 못함
- **적합**: 1차 탐색용. "이 거래처에 CTP를 대략 얼마에 팔고 있지?"

### Approach B: 거래처×항목×대수 3차원 매트릭스
- **장점**: 부수 구간별 단가 변동까지 분석 가능
- **단점**: 데이터가 얇은 구간에서 신뢰도 저하
- **적합**: 2차 정밀 분석용 (v2)

**선택 근거**: 우선 전체 그림을 그리는 것이 목적. 정밀 분석은 나중에.

## 4. 기술 설계

### 4-1. 파일 구조

```
견적계산기/
├── scripts/
│   ├── reverse_engineer_pricing.py   ← 핵심 스크립트
│   ├── standard_pricing.json         ← 갑우 표준단가표 (항목코드 → 단가)
│   └── output/
│       ├── pricing_matrix.csv        ← 거래처×항목 단가 집계
│       ├── pricing_matrix.json       ← 대시보드용 JSON
│       └── dashboard.html            ← 단일 파일 HTML 대시보드
```

### 4-2. Python 스크립트 처리 흐름

```
1. 엑셀 63,523행 로드 (pandas, 읽기 전용)
2. 필터: 단가 > 0인 행만 (약 59,291건)
3. (거래처, 항목) GROUP BY → 집계:
   - median (중앙값) — 대표 단가
   - mode (최빈값) — 가장 많이 적용한 단가  
   - min, max — 단가 범위
   - count — 건수
   - nunique — 고유 단가 수 (1이면 고정 단가)
4. standard_pricing.json 로드 → 항목코드 매칭
5. 차이(%) = (실제 중앙값 - 표준단가) / 표준단가 × 100
6. CSV + JSON 출력
```

### 4-3. 표준단가 매칭

갑우 표준단가표에서 항목코드별 기준 단가를 추출:
- 703002 (CTP) → 7,000원
- 303001 (인쇄) → 부수 구간별 상이 (대표값 사용)
- 기타 후가공 → 공정별 고정가

표준단가표에 매칭되지 않는 항목은 차이(%) 컬럼에 "N/A" 표시.

### 4-4. HTML 대시보드 사양

- **단일 HTML 파일** (외부 의존성 없음, 로컬에서 바로 열기)
- **기능**:
  - 거래처 검색/필터
  - 항목별 정렬 (단가 높은순/낮은순/건수순)
  - 표준단가 대비 차이 색상 코딩 (할인=파랑, 할증=빨강)
  - 거래처 클릭 시 해당 거래처의 전체 항목 상세 펼침
- **데이터 로딩**: JSON을 HTML 내부에 인라인 삽입 (별도 서버 불필요)

### 4-5. 라벨링 원칙

모든 출력물에 다음 라벨 필수 표시:
> **초안(ERP 역산)** — 영업자 확인 전. 실제 단가표와 대조 필요.
