## 2. 여러 에이전트가 동시에 지적한 "진짜 치명" 3건 (우선순위 최상)

### 🔴 TOP-1. 배치 실패 알림 부재 (QA + Infra + Code 3명 동시 지적)

**현상**: `daily_update.sh`가 실패해도 log 파일에만 기록. 사용자가 매일 log를 열어보지 않으면 수일간 실패를 모름. 대시보드는 "전날 데이터"를 정상처럼 보여줌.

**최악 시나리오**: Jack이 주말 출장 중 배치 3일 실패 → 월요일 대표 회의에서 "금요일 실적" 자료로 의사결정 → 실제는 목요일 데이터 → 수치 오류 발각 → 대시보드 전체 신뢰 붕괴.

**조치안** (반나절):
1. `daily_update.sh`에 Slack Webhook 호출 추가 (성공/실패 모두)
2. Supabase에 `sync_log` 테이블 신설. 배치 종료 시 `{run_at, rows_inserted, status, error}` 1행 기록
3. 대시보드 상단에 "데이터 기준: YYYY-MM-DD HH:MM" 고정 노출
4. 24시간 이상 미갱신 시 빨간 배지 자동 표시

### 🔴 TOP-2. ETL 라인 데이터 손실 위험 (Code 지적, Data 정합성)

**위치**: [sync_erp_to_supabase.py:252-302](scripts/sync_erp_to_supabase.py#L252) `replace_lines()` 함수

**문제**: `erp_quote_lines` 삭제 → 새 라인 삽입을 **2단계로** 수행하는데 트랜잭션 경계가 없음. DELETE 성공 후 INSERT 중 실패 시(`sys.exit(1)`) **라인 데이터가 영구 손실**. 복구 불가능.

**조치안** (1일):
- 옵션 A: PostgreSQL RPC 함수로 감싸서 트랜잭션 보장 (권장)
- 옵션 B: staging 테이블에 전부 insert → 성공 시 swap (운영 영향 0)

### 🔴 TOP-3. 스펙 내부 모순 — KPI 번호 불일치 (Design-Validator 지적)

**문제**: `gmd_v1_spec.md`의 KPI 번호와 `kpi_v1_pilot_run.txt` 검증 결과의 KPI 번호가 어긋남. 두 개발자가 스펙을 읽으면 다르게 구현할 수 있음.

**예**: 스펙 KPI-5(이탈 위험) = pilot_run의 KPI-6. 스펙 KPI-6(용지 매입) = pilot_run의 KPI-8.

**조치안** (30분): 스펙에 "이 문서의 KPI 번호 기준이 공식. pilot_run의 번호는 탐색 시점 임시 번호" 주석 추가 + 검증값 섹션에 교차 참조표 삽입.

## 7. 결정 필요 사항 (Jack)

이 리포트 반영 시 **스펙 문서를 수정해야 할 곳이 6군데**입니다. 크게 3가지 결정이 필요해요.

### Q1. "이번달 예상 매출" 계산식을 어느 것으로?
- (a) MTD × (30/경과일) 선형 연장 — 단순하지만 월초에 큰 오차
- (b) 전년 동월 × (YoY 성장률) — 안정적이지만 작년 데이터 필요
- (c) "예상" 빼고 "MTD 누계"만 표시 — 가장 안전
- **제 추천: (c) MTD 누계 + 작은 글씨로 "월말 예상 범위: {작년 동월×0.9} ~ {작년 동월×1.1}" 병기**

### Q2. 영업자 실적(KPI-3) 노출 범위를 지금 결정?
- (a) v1에는 모든 approved_users에게 공개 (현재 설계) → 법적 여지 + 영업자 반발 가능
- (b) v1부터 경영진 전용 패널로 분리 (RBAC 추가 공사 0.5일)
- (c) 실명 대신 "영업 1팀 1번" 같은 익명화 (0.5일)
- **제 추천: (b) — 지금부터 분리하는 게 나중에 돌리는 것보다 쉽습니다**

### Q3. Go-Live 일정을 4/25로 연기?
- (a) 원래대로 4/22 고수 — 위 치명 결함 3건을 커버 못 함
- (b) 4/25로 연기 (+3일) — 보완 완료 후 안전 오픈
- **제 추천: (b)**

## 8. 이 리포트를 쓴 이유 (사용자 지시)

사용자가 `/bkit:bkit`로 "CTO에서 에이전트 8명 이상 불러서 동시 검증 → 문서 작성 → 작업" 순서를 지시함. 단독 에이전트 리뷰로는 놓쳤을 **다섯 가지 중요 발견**이 병렬 검토에서 드러났습니다.

4. Infra의 Jack Mac SPOF와 Free tier 한계 — 9개월 내 재고해야 할 구조 문제
5. Gap-Detector의 완성도 35% 수치 — 남은 작업량의 객관적 측정
